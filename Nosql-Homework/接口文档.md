# Nosql-Homework 接口文档

> GitHub 开源项目分析平台 — 后端 API 使用说明

---

## 1. 项目简介

基于 Spring Boot + MongoDB 的 GitHub 开源项目数据分析平台。支持按语言/品类/星标等条件检索项目、自然语言语义搜索、爬虫任务控制等功能。

**技术栈**: Java 21 · Spring Boot 4.0 · MongoDB · LangChain4j (LLM 意图解析)

---

## 2. 通用约定

### 2.1 基础地址

```
http://localhost:8080
```

### 2.2 统一响应格式

所有接口返回 JSON，结构如下：

```json
{
  "code": 0,        // 0=成功, 非0=失败
  "msg": "ok",      // 提示信息
  "data": { ... }   // 业务数据（可能为 null）
}
```

### 2.3 分页响应格式

涉及列表的接口，`data` 为分页对象：

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "total": 900,    // 总记录数
    "page": 1,       // 当前页码 (1-based)
    "size": 20,      // 每页条数
    "records": [...] // 数据数组
  }
}
```

### 2.4 项目 (Project) 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String | MongoDB _id |
| `githubId` | Long | GitHub 仓库 ID |
| `name` | String | 项目名 |
| `fullName` | String | owner/repo 格式 |
| `description` | String | 项目描述 |
| `language` | String | 主编程语言 |
| `topics` | Array | GitHub 话题标签 |
| `category` | String | 软件品类，见 §2.5 |
| `starsCount` | Integer | Star 数 |
| `forksCount` | Integer | Fork 数 |
| `commitsCount` | Integer | 提交数 |
| `contributorsCount` | Integer | 贡献者数 |
| `qualityScore` | Double | 质量评分 |
| `longTermValue` | Double | 长期价值评分 |
| `createdAt` / `updatedAt` | Date | GitHub 时间 |
| `crawledAt` | Date | 采集时间 |

### 2.5 品类 (category) 说明

通过关键词规则自动分类，基于 GitHub topics 标签 + description 加权匹配：

| 分类值 | 涵盖内容 |
|--------|----------|
| `ai` | 机器学习、深度学习、LLM、NLP、CV |
| `web` | 前端框架、网站、SPA、SSR |
| `mobile` | Android、iOS、Flutter、React Native |
| `desktop` | Electron、Tauri、Qt、WPF 等桌面应用 |
| `framework` | Spring、Django、Express 等后端框架 |
| `library` | SDK、工具库、驱动、插件 |
| `cli` | 命令行工具、终端应用 |
| `api` | REST API、GraphQL、gRPC |
| `database` | 数据库、ORM、缓存 |
| `devops` | Docker、K8s、CI/CD、监控 |
| `security` | 加密、认证、安全工具 |
| `game` | 游戏引擎、游戏开发 |
| `tool` | 开发工具、脚手架、生成器 |
| `data` | 数据分析、大数据、ETL |

> `null` = 从未分类 | `""` (空串) = 已尝试但未匹配到任何分类

## 3. 项目接口

### 3.1 项目列表（分页+筛选）

```
GET /api/projects
```

**请求参数**:

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `page` | int | 否 | 1 | 页码 |
| `size` | int | 否 | 20 | 每页条数 |
| `language` | String | 否 | — | 语言过滤，如 `Java`、`Python` |
| `category` | String | 否 | — | 品类过滤，如 `ai`、`web` |
| `minStars` | int | 否 | — | 最低星标数 |
| `sortBy` | String | 否 | `updatedAt` | 排序字段：`starsCount` / `forksCount` / `updatedAt` |

**示例**:

```bash
# 查询 AI 品类的 Java 项目，至少 100 星，按星标降序
curl "http://localhost:8080/api/projects?language=Java&category=ai&minStars=100&sortBy=starsCount&page=1&size=10"
```

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "total": 42,
    "page": 1,
    "size": 10,
    "records": [
      {
        "id": "6a315dcbf383b51e564f3e5e",
        "fullName": "Snailclimb/JavaGuide",
        "description": "Java 面试指南...",
        "language": "Java",
        "category": "ai",
        "starsCount": 156405,
        "topics": ["ai", "java", "mcp", "deepseek", "springai"]
      }
    ]
  }
}
```

### 3.2 项目详情

```
GET /api/projects/{id}
```

### 3.3 项目 Commits（分页）

```
GET /api/projects/{id}/commits?page=1&size=20
```

### 3.4 项目贡献者

```
GET /api/projects/{id}/contributors
```

### 3.5 按需拉取 Commits

项目初次访问时 commits 可能为空，调用此接口从 GitHub API 拉取：

```
POST /api/projects/{id}/fetch-commits
```

```json
{
  "code": 0,
  "msg": "ok",
  "data": { "projectId": "6a31...", "fetched": 300 }
}
```

### 3.6 语言分布统计

```
GET /api/projects/language-stats
```

```json
{
  "code": 0,
  "data": [
    { "language": "JavaScript", "count": 230 },
    { "language": "Python", "count": 185 },
    { "language": "Java", "count": 152 }
  ]
}
```

### 3.7 新增项目

```
POST /api/projects
Content-Type: application/json

{ ... Project 对象 ... }
```

### 3.8 删除项目

```
DELETE /api/projects/{id}
```

## 5. 自然语言搜索

### 5.1 语义检索

```
POST /api/search/nl
Content-Type: application/json
```

**请求体**:

```json
{
  "query": "找 Java 语言的高星标 Web 框架项目",
  "topK": 10
}
```

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `query` | String | 是 | — | 自然语言查询 |
| `topK` | int | 否 | 10 | 返回数量 |

**LLM 意图解析能力**:

系统会将自然语言解析为结构化过滤条件：

| 用户说法 → | 解析效果 |
|-------------|----------|
| "Python 机器学习" | language=Python, keywords=machine learning, category=ai |
| "高星标 Web 框架" | sortBy=stars, minStars=1000, category=web |
| "最近更新的 Go 项目" | language=Go, sortBy=updated |
| "热门的 CLI 工具" | sortBy=stars, minStars=500, category=cli |
| "数据库相关的 Java 库" | language=Java, keywords=database, category=library |

**提示词技巧**:

- 提语言：`Java`、`Python`、`Go` 等 → 精确匹配 `language` 字段
- 提品类：`Web 框架`、`AI`、`命令行`、`游戏` 等 → 匹配 `category` 字段
- 提热度：`高星标` → ≥1000 星、`热门` → ≥500 星、`顶级` → ≥5000 星
- 提功能：`数据库`、`机器学习`、`认证` 等 → 作为 content keywords 匹配 description

**响应**:

```json
{
  "code": 0,
  "msg": "ok",
  "data": {
    "query": "找 Java 语言的高星标 Web 框架项目",
    "parsedIntent": "project_search",
    "language": "Java",
    "category": "web",
    "keywords": "framework web",
    "items": [
      {
        "id": "665f9b2f...",
        "fullName": "spring-projects/spring-boot",
        "description": "Spring Boot helps you to create...",
        "language": "Java",
        "starsCount": 78000,
        "matchScore": 0.95
      }
    ]
  }
}
```

**查询策略（自动选择）**:

| 条件组合 | 策略 | 性能 |
|----------|------|------|
| language + keywords + minStars | 语言+星标过滤 → 内存关键词匹配 | ⭐⭐⭐ |
| language + minStars | 语言+星标直查 | ⭐⭐⭐⭐ |
| language only | 语言过滤 | ⭐⭐⭐⭐ |
| keywords only | description/name 模糊匹配 | ⭐⭐ |
| category only | 品类精确过滤 | ⭐⭐⭐⭐ |
| 兜底 | 全表按星标降序 | ⭐ |

### 5.2 搜索历史

```
GET /api/search/history?page=1&size=20
```

每次调用 `/api/search/nl` 会自动记录搜索历史。

## 6. 爬虫任务接口

> 所有爬虫接口均为异步执行，返回即表示任务启动，实际执行日志在控制台查看。

### 6.1 按语言采集

```
POST /api/crawler/trigger?language=Java&maxPages=5
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `language` | Java | GitHub 语言名 |
| `maxPages` | 5 | 翻页数（每页 30 条） |

> 按 GitHub 星标降序拉取，自动去重入库，新建项目自动打品类分类。

### 6.2 增量回填 Contributors

```
POST /api/crawler/backfill/missing
```

只补充缺失 contributors 数据的项目。

### 6.3 全量回填 Contributors

```
POST /api/crawler/backfill/all
POST /api/crawler/backfill/all?force=true
```

`force=true` 强制覆盖已有数据。

### 6.4 语言名归一化

```
POST /api/crawler/normalize-languages
```

修复历史数据中 `js` / `javascript` / `JavaScript` 等不一致问题。

### 6.5 回填项目分类

```
POST /api/crawler/backfill-categories
POST /api/crawler/backfill-categories?force=true
```

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `force` | false | `true` 时对已标记为 `""` 的项目重新尝试分类 |

响应示例：

```json
{
  "code": 0,
  "data": {
    "total": 900,
    "classified": 612,
    "unclassified": 288,
    "retried": 0,
    "force": false
  }
}
```

## 7. 典型使用流程

### 7.1 首次启动

```bash
# 1. 采集数据（每页 30 条，5 页共 150 条）
curl -X POST "http://localhost:8080/api/crawler/trigger?language=Java&maxPages=5"
curl -X POST "http://localhost:8080/api/crawler/trigger?language=Python&maxPages=5"
curl -X POST "http://localhost:8080/api/crawler/trigger?language=JavaScript&maxPages=5"

# 2. 补充 contributors
curl -X POST "http://localhost:8080/api/crawler/backfill/missing"

# 3. 查看语言分布
curl "http://localhost:8080/api/projects/language-stats"

# 4. 浏览项目
curl "http://localhost:8080/api/projects?language=Java&minStars=500&page=1"
```

### 7.2 日常使用

```bash
# 语义搜索
curl -X POST "http://localhost:8080/api/search/nl" \
  -H "Content-Type: application/json" \
  -d '{"query":"Python 深度学习框架","topK":10}'

# 品类筛选
curl "http://localhost:8080/api/projects?category=ai&sortBy=starsCount"

# 查看项目详情
curl "http://localhost:8080/api/projects/6a315dcbf383b51e564f3e5e"

# 拉取 commits（首次访问某项目时）
curl -X POST "http://localhost:8080/api/projects/6a315dcbf383b51e564f3e5e/fetch-commits"
```

## 8. 品类分类关键词扩展

如需新增或调整分类规则，编辑 `CategoryClassifier.java` 中的 `CATEGORY_KEYWORDS` Map。关键词全小写，topics 匹配权重 3x，description 匹配权重 1x。

新增后重启服务，运行回填：

```bash
curl -X POST "http://localhost:8080/api/crawler/backfill-categories?force=true"
```
